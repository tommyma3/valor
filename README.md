# VALOR: Value-Guided Context Management for AI Agents

This repository implements the pipeline described in `idea.md`:

- MDP state = (question, evolving memory, previous tool query/result)
- Action = (Think, memory update, next tool query)
- Reward = 1 for correct final answer, 0 otherwise
- Value head learns a binary distribution over returns
- Policy is trained with an advantage-conditioned indicator

The implementation mirrors the IterResearch flow (policy step -> environment step -> transition) while keeping the code style close to the CS185/285 homework structure.

## Repository layout

- `prompts/prompts.py`: IterResearch prompt templates (source of system prompt)
- `valor/system_prompts.py`: loader for the system prompt used by VALOR
- `valor/prompts.py`: prompt templates and action parsing
- `valor/model.py`: shared transformer + policy head + value head
- `valor/data.py`: datasets and collators
- `valor/trajectory.py`: return computation and value label helpers
- `scripts/collect_trajectories.py`: run the policy to produce actions
- `scripts/compute_rewards.py`: assign binary rewards
- `scripts/train_value.py`: train value head
- `scripts/compute_advantages.py`: compute advantages + labels
- `scripts/train_policy.py`: train policy with QLoRA advantage conditioning
- `scripts/generate_sft.py`: generate SFT data via Kimi API
- `scripts/run_iterresearch.py`: run the IterResearch prompt on a local model
- `scripts/test_llm.py`: quick generation sanity check

## Setup (uv)

Create a Python 3.12 environment with `uv` and install dependencies.

```bash
uv venv --python 3.12
uv sync
```

The repository is configured to install PyTorch from the CUDA 13.0 wheel index (`cu130`).
If you need a different CUDA version, update the `torch` index in `pyproject.toml` and re-lock.

```bash
uv lock --upgrade
uv sync
```

Note: `uv.lock` is generated by `uv` and should not be edited manually.

## Data format

All scripts use JSONL. Each line is one transition.

### State-only records (for collection)

```json
{"trajectory_id":"traj-0001","t":0,"question":"...","memory":"...","prev_tool_query":"","prev_tool_result":""}
```

### Trajectory records (for training)

```json
{
  "trajectory_id": "traj-0001",
  "t": 0,
  "question": "...",
  "memory": "...",
  "prev_tool_query": "...",
  "prev_tool_result": "...",
  "action_think": "...",
  "action_memory_update": "...",
  "action_tool_query": "...",
  "reward": 0,
  "return": 1,
  "value_pred": 0.42,
  "advantage": 0.58,
  "advantage_label": 1
}
```

Notes:
- `reward` should be 1 only for the final transition of a trajectory.
- `return` is computed by `scripts/compute_advantages.py` or `valor/trajectory.py`.
- `advantage_label` is the binary indicator used for the advantage-conditioned policy.

## Prompt format

The policy is trained to output a structured action:

```
<THINK>
...
</THINK>
<MEMORY>
...
</MEMORY>
<TOOL>
...
</TOOL>
```

The system prompt is loaded from `prompts/prompts.py` via `valor/system_prompts.py`.

## Generate SFT data (Kimi)

The SFT generator calls the Kimi API with an OpenAI-compatible Chat Completions request.
Set your API key and discover available models first.

```bash
export MOONSHOT_API_KEY=...  # or set it in your server environment
uv run python scripts/generate_sft.py --list-models
```

Then generate SFT trajectories:

```bash
uv run python scripts/generate_sft.py \
  --input data/sft_states.jsonl \
  --output data/sft.jsonl \
  --model <YOUR_KIMI_MODEL_ID>
```

Input format for `sft_states.jsonl` is the same as the state-only records shown above.

## Run with SGLang (no training)

If you already have a SGLang server running, point `collect_trajectories.py` at it instead of loading the model locally.

```bash
uv run python scripts/collect_trajectories.py \
  --states data/states.jsonl \
  --output data/trajectories.jsonl \
  --checkpoint Qwen/Qwen3.5-35B-A3B \
  --sglang-url http://127.0.0.1:8000 \
  --sglang-model Qwen/Qwen3.5-35B-A3B
```

Optionally set `SGLANG_API_KEY` if your server requires auth.

## Pipeline

### 1) (Optional) Pretrain with IterResearch-style SFT

Use `scripts/train_policy.py` with `--alpha 0` to warm-start the model on supervised data. The policy trainer now uses QLoRA by default for Qwen3.5-35B-A3B.

```bash
uv run python scripts/train_policy.py \
  --data data/sft.jsonl \
  --output checkpoints/policy_sft \
  --backbone Qwen/Qwen3.5-35B-A3B \
  --alpha 0
```

### 2) Collect trajectories

```bash
uv run python scripts/collect_trajectories.py \
  --states data/states.jsonl \
  --output data/trajectories.jsonl \
  --checkpoint checkpoints/policy_sft
```

### 3) Compute rewards

By default this uses `final_answer` vs `gold_answer` on the final step in each trajectory.

```bash
uv run python scripts/compute_rewards.py \
  --data data/trajectories.jsonl \
  --output data/trajectories_rewarded.jsonl
```

### 4) Train value head

```bash
uv run python scripts/train_value.py \
  --data data/trajectories_rewarded.jsonl \
  --output checkpoints/value
```

### 5) Compute advantages

```bash
uv run python scripts/compute_advantages.py \
  --data data/trajectories_rewarded.jsonl \
  --value-model checkpoints/value \
  --output data/trajectories_adv.jsonl
```

### 6) Train policy with advantage conditioning

```bash
uv run python scripts/train_policy.py \
  --data data/trajectories_adv.jsonl \
  --output checkpoints/policy_adv \
  --alpha 1.0 \
  --indicator-drop-prob 0.1 \
  --device-map auto
```

## Quick LLM test

Once a checkpoint exists, run a single generation to confirm the model responds.

```bash
uv run python scripts/test_llm.py \
  --checkpoint checkpoints/policy_adv \
  --question "Summarize the key idea in one sentence."
```

## Implementation notes

- The value head is a 2-class classifier with logits for `p(V=0|s)` and `p(V=1|s)`.
- The policy and value heads share the same transformer backbone.
- Advantage conditioning is implemented by adding a textual indicator (`Advantage: positive/negative`) to the prompt.
- Policy training uses 4-bit QLoRA adapters on top of the Qwen3.5-35B-A3B backbone.
- The default LoRA targets cover both attention projections and MLP/expert projections: `q_proj,k_proj,v_proj,o_proj,gate_proj,up_proj,down_proj,w1,w2,w3`.
- When `--indicator-drop-prob` is non-zero, some samples are trained without the indicator to reduce forgetting.
- Use `--freeze-backbone` in `scripts/train_value.py` if you only want to learn the value head.

## Customizing the reward function

The reward script is intentionally simple. For your environment, replace the logic in `scripts/compute_rewards.py` to call your evaluator or environment API and write binary rewards to each trajectory step.

## Large model usage

Policy training now assumes QLoRA for `Qwen/Qwen3.5-35B-A3B`. Install `bitsandbytes` alongside `torch`, `transformers`, and `peft` on the remote server, and use `--device-map auto` when the adapter training needs multiple GPUs. Saved policy checkpoints are PEFT adapter directories, and the local inference scripts automatically reconstruct `base model + adapter` when loading them.
